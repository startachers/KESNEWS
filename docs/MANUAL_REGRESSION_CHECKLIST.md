## 1. 사전 준비

- [ ] `.DS_Store`를 삭제했다.
- [ ] `.gitignore`가 적용됐다.
- [ ] `git status --short`에서 의도한 파일만 보인다.
- [ ] 원본 HTML을 수정하지 않았다.
- [ ] 가능하면 `?noauto`로 자동 네트워크 검색을 막고 기준 화면을 열었다.
- [ ] 개발자도구 Console을 열었다.
- [ ] 분리본은 로컬 정적 서버로 열었다. ES Module은 `file://`에서 로드되지 않는다. 예: 저장소 루트에서 `python3 -m http.server 8000` 실행 후 `http://127.0.0.1:8000/frontend/index.html` 접속.
- [ ] 원본(`file://`)과 분리본(`http://`)의 실행 환경 차이를 결과 기록에 남겼다.

## 2. 초기 화면

- [ ] 원본이 오류 없이 열린다.
- [ ] 분리본이 오류 없이 열린다.
- [ ] 헤더, KPI, 요약, Top Issues, 기사목록, 우측 패널 배치가 동일하다.
- [ ] 반응형 폭에서 주요 레이아웃이 동일하다.
- [ ] Console에 신규 오류가 없다.

## 3. 샘플·기사 목록

- [ ] 샘플 화면을 불러올 수 있다.
- [ ] 기사 제목·매체·배지·요약이 동일하다.
- [ ] 검색어 필터가 동일하게 작동한다.
- [ ] 분류·위험도·선정 필터가 동일하다.
- [ ] 관련도순·위험도순·최신순·매체순 결과가 동일하다.

## 4. 편집 상태

- [ ] 기사 선택 체크가 저장된다.
- [ ] 기사 선택 해제가 저장된다.
- [ ] Gemma 추천 적용 후 Media Coverage에서 일반·관련 기사를 추가 선정·해제할 수 있다.
- [ ] 펼친 관련기사의 `묶음에서 제거`가 기사 원본·선정·메모를 보존한다.
- [ ] Top Issues 카드 우측 상단 `탑이슈 제거`가 기사 선정은 유지하고 Top 태그만 해제한다.
- [ ] 중요 표시가 저장된다.
- [ ] 기사 메모가 저장된다.
- [ ] 새로고침 후 위 상태가 동일하다.
- [ ] 휴지통 동작이 원본과 동일하다. 장기 보존 의미는 Phase 4에서 수정한다.

## 5. 브리핑·날짜

- [ ] 작성 부서·담당 입력이 저장된다.
- [ ] 기본 요약 생성 결과가 동일하다.
- [ ] 요약 직접 수정이 저장된다.
- [ ] CEO 참고·지시사항이 저장된다.
- [ ] 날짜 변경 후 날짜별 저장본이 구분된다.
- [ ] 오늘 날짜로 돌아왔을 때 원래 상태가 복원된다.

## 6. 설정·수집

- [ ] 설정 modal이 열린다.
- [ ] 검색식·키워드·수집상한 설정을 저장할 수 있다.
- [ ] 기본값 복원이 동일하다.
- [ ] 네트워크 수집을 실행했다면 provider별 성공·실패 표시가 원본과 동일하다.
- [ ] 일부 provider 실패 시 현행 데이터 교체 위험을 `KNOWN_RISKS.md`의 LEG-001로 기록했다.

## 7. AI

- [ ] AI 세션 토큰 전달(meta 태그 `kesco-ai-token` 또는 URL `#ai=` hash)이 분리본에서도 동일하게 동작한다.
- [ ] AI 서버 오프라인 표시가 동일하다.
- [ ] Ollama와 실행 도우미가 준비된 환경에서는 health 확인이 동일하다.
- [ ] 기사 선정 후 분석 버튼 상태가 동일하다.
- [ ] AI 결과·stale 표시·담당자 수정 상태가 동일하다.
- [ ] `ChatGPT 바로가기`는 ChatGPT를 새 창으로 열고 분석 프롬프트를 클립보드에 복사한다.
- [ ] `Claude 바로가기`는 Claude를 새 창으로 열고 분석 프롬프트를 클립보드에 복사한다.
- [ ] 두 바로가기는 MD를 자동 생성·다운로드·첨부·전송하지 않으며 팝업 또는 클립보드 차단을 오류로 표시한다.
- [ ] 근거 ID 검증은 Phase 7 항목이며 Phase 1에서 추가하지 않았다.

## 8. 가져오기·내보내기

- [ ] JSON을 내보낼 수 있다.
- [ ] JSON을 다시 가져올 수 있다.
- [ ] 현행 왕복 누락을 `KNOWN_RISKS.md` LEG-004로 기록했다.
- [ ] CSV를 내보낼 수 있다.
- [ ] CSV를 다시 가져올 수 있다.
- [ ] 현행 enum·분류 왕복 차이를 LEG-005로 기록했다.

## 9. 인쇄

- [ ] 인쇄 미리보기가 열린다.
- [ ] 편집 컨트롤이 인쇄에서 숨겨진다.
- [ ] 미선정 기사가 인쇄에서 제외된다.
- [ ] A4 페이지 나눔과 한글 줄바꿈이 원본과 실질적으로 동일하다.

## 10. 기상 기반 선제대응

- [ ] `KMA_SERVICE_KEY` 미설정 시 설정 필요 상태를 표시하고 기존 기사 브리핑은 계속 사용할 수 있다.
- [ ] 메인 언론브리핑에는 `기상 특이사항`의 상태·특보·주요 예보와 최저·최고기온, 시간당 예상 강수량, 최고 강수확률이 표시된다.
- [ ] 기상 핵심 항목은 아이콘과 텍스트가 함께 표시되며 색상만으로 상태를 구분하지 않는다.
- [ ] 특보마다 영향 권역이 표시되고 현재특보·예비특보가 구분되며, 최고 위험 지역은 시·군 단위 `최우선 확인`으로 표시된다.
- [ ] `상세 기상정보 · 관리`를 누르면 모달이 열리고 닫기 버튼·배경·Escape로 닫힌다.
- [ ] 상세 모달에 최대 시간강수량·최고기온·최우선 지역과 특보·단기·중기예보의 발표시각·오류 상태가 표시된다.
- [ ] CEO 미리보기·인쇄에는 `midForecast: partial` 같은 provider명·내부 상태값·오류 원문이 표시되지 않는다.
- [ ] 상세 모달을 열면 `종합판단` 탭만 보이고 `특보·대응`, `권역비교` 탭 전환 시 해당 정보만 표시된다.
- [ ] 상세 모달에서 현재·예비특보 공식 원문과 발효 지역·발표/발효시각을 확인할 수 있다.
- [ ] 비교 날짜 변경 시 6개 권역의 기온·강수확률·시간강수량·최대풍속이 해당 날짜 값으로 바뀐다.
- [ ] 공식 경보와 강수량·확률·기온 임계값에 해당하는 일자만 단계별로 강조되고 텍스트 배지가 함께 표시된다.
- [ ] 권역비교에서 선택 날짜의 항목별 최댓값과 고위험 권역이 다른 권역보다 우선적으로 식별된다.
- [ ] 상세 모달에는 선택 권역의 모든 위험 신호와 공식 근거·발효시각이 표시된다.
- [ ] 기상 새로고침 중 기존 정상 context가 사라지지 않는다.
- [ ] 단기예보·중기예보·특보의 정상·부분 실패·stale·unknown 상태가 구분된다.
- [ ] 보고일 포함 D0~D6의 7일 카드가 표시된다.
- [ ] 전국·수도권·강원권·충청권·호남권·영남권·제주권 전환 시 예보와 위험 신호가 해당 권역으로 바뀐다.
- [ ] 기상 위험 신호별 보고 반영 체크, 담당자 단계, 메모가 저장되고 새로고침 후 유지된다.
- [ ] 검토 완료 전 기상정보가 CEO 보고와 Gemma 입력에 반영되지 않는다.
- [ ] 검토 완료 후 최신 context가 생겨도 기존 검토 context가 자동 교체되지 않고 경고가 표시된다.
- [ ] 기상정보 보고 반영 해제 후 신호 선택·담당자 단계·메모가 보존된다.
- [ ] `critical`·`watch` 상태가 색상뿐 아니라 텍스트로 식별되고 키보드로 편집할 수 있다.
- [ ] CEO 미리보기와 최종 HTML의 기상 context·선택 신호·담당자 override가 동일하다.
- [ ] CEO 미리보기의 기상 요약은 1페이지 하단에서 가장 중요한 현상 1건만 `(폭우) 7. 19~20 최대 시간당 50mm · 일 최대 200mm / 충청·호남 · 우려: 저지대 전기설비 침수·누전·감전` 형식으로 표시하고, 차순위 현상·강수확률·종합 위험도·요일별 카드를 표시하지 않는다.
- [ ] AI 기상 문장의 `Wxx` 근거가 기사 `Axx`와 분리되고, 잘못된 기상 근거는 서버 템플릿으로 대체된다.
- [ ] JSON 내보내기→가져오기에서 기상 context·검토 상태·신호 override가 보존된다.
- [ ] 편집 화면 인쇄에서는 기상 편집 모달이 숨겨지고, 최종 보고의 7일 요약과 위험 신호가 A4 폭을 벗어나지 않는다.
