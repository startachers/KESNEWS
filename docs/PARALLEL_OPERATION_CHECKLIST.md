기존 브리핑 절차를 기준선으로 유지하면서 새 앱 결과를 비교한다. 차이가 있으면 새 앱 데이터를

## 매일

- [ ] 로그인 뒤 서버가 자동 시작되고 health의 DB 상태가 정상이다.
- [ ] 2시간 자동수집 이력이 있으며 실패 시 마지막 정상 수집과 기존 후보가 유지된다.
- [ ] 기존 절차 대비 중대한 기사 누락이 없다. 누락 원인과 provider를 기록했다.
- [ ] 기사 선택·해제, 중요 표시, 메모가 새로고침 뒤 유지된다.
- [ ] 담당자 수정 요약이 재수집·AI 실패 뒤에도 유지된다.
- [ ] 최종 version의 DB snapshot, JSON, HTML version이 일치한다.
- [ ] app/collection/ai 로그에 원문·비밀값이 없고 운영 오류를 확인할 수 있다.

## 기간 중 1회 이상

- [ ] 앱을 두 번 실행해 두 번째 서버가 뜨지 않는지 확인했다.
- [ ] 네트워크 차단 또는 provider fixture 실패에서 기존 후보 보존을 확인했다.
- [ ] Ollama 중지 상태에서 기존 AI 결과·수동 요약 보존을 확인했다.
- [ ] 별도 임시 DB로 최근 백업 복구를 연습하고 무결성·최종 version을 비교했다.
- [ ] 로그 회전 설정(5 MiB, 과거 5개)을 확인했다.
- [ ] 재부팅 뒤 server agent와 다음 자동수집을 확인했다.
- [ ] JSON·CSV 내보내기와 JSON 복원, 인쇄 미리보기를 확인했다.
