# 수동 기사 본문 입력 회귀 기록 (2026-07-23)

## 변경 범위

- 관련기사 행의 `본문 직접 입력` 편집 UI
- `PUT /api/articles/{article_id}/manual-body`
- 자동 원문과 분리된 `article_body_overrides` 저장소 및 `manual_paste` 품질 이력
- 수동 본문의 대표기사 판정·AI 근거 우선 사용과 JSON schema 13 백업/복원

## 자동 검증

- 수동 본문 저장 후 품질 적격 및 대표기사 지정 가능
- 이후 자동 재추출이 실패해도 수동 본문과 적격 판정 유지
- 최종 확정 작업본 수정 차단 및 보고일 기사 범위 확인
- migration 신규 설치·재실행 안전성
- JSON 내보내기·가져오기에서 수동 본문 override 복원

## 수동 회귀 결과

- [x] 임시 DB 로컬 브라우저에서 편집기 열기와 저장 확인
- [x] 688자 정상 본문 저장 후 `수동 본문` 배지, 83점 적격 판정과 자동 대표 승격 확인
- [ ] Console 신규 오류 확인
- [ ] 잘린 본문 및 Google 뉴스 주소 사용 시 차단 사유 확인
- [ ] 기사 선택·해제, 중요 표시, 메모, 날짜 변경, 요약 직접 수정 확인
- [ ] JSON·CSV 내보내기와 인쇄 미리보기 확인

브라우저 상호작용이 필요한 항목은 작업 환경에서 실제 운영 데이터 변경 없이 자동화할 수 없어
미수행으로 남긴다. 자동 테스트는 외부 네트워크를 사용하지 않았다.

## 복구

작업 전 정식 DB 백업을 복원하는 방법을 우선한다. 빈 수동 본문 이력만 되돌릴 때는 서버를
중지하고 `article_body_overrides`를 drop한 뒤 `schema_migrations`에서
`0028_manual_article_body_override.sql` 행을 제거한다. 이 테이블은 자동 수집 원문
`articles.body_text`를 변경하지 않으므로 기존 자동 원문은 그대로 남는다.
