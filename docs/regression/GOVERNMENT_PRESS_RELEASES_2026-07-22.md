# 정부부처 보도자료 수집·필터 회귀검증 (2026-07-22)

## 변경 범위

- 정책브리핑 보도자료 OpenAPI의 날짜 요청·XML 응답 파싱
- 정부부처 observation 기반 `governmentPressRelease` 조회 필드
- Media Coverage의 `전체 자료 / 정부부처 보도자료 / 언론기사` 필터
- 정부부처 보도자료 카드의 출처 배지
- 정부부처 수집 자료의 관련도·제외어 예외와 24시간 후보 제한 유지
- `오늘 기사 검색`과 별도 `정부 보도자료 수집` 버튼·API

기존 브리핑 기사 PATCH, AI 분석, 최종 snapshot/CEO 보고 계약은 변경하지 않았다.

### 운영 재검증 (17:53 KST)

- `articles.content_key` UNIQUE 충돌을 재현해 content key 선조회 병합으로 수정했다.
- 전체 자동검증 `396 passed`, `ruff check .` 통과.
- 운영 서버 재시작 후 오늘 기사 검색은 `partial`로 정상 완료됐다.
- 최신 실행은 원본 1,679건, 중복 정리 후 234건(신규 8건, 기존 병합 226건)으로 완료됐다.
- 정부부처 자료는 관련도·제외어와 무관하게 유지하되 24시간 범위는 적용했다. 최종 후보 7건은
  기후에너지환경부 6건과 국무조정실 1건이다.
- 국무조정실 50건·기후에너지환경부 10건은 provider 성공으로 기록됐다.
- 정책브리핑 OpenAPI는 동일 인증키로 HTTP/HTTPS 및 User-Agent를 달리해도 본문 `Forbidden`,
  HTTP 403을 반환했다. 키 형식은 decoding 값으로 확인됐으므로 공공데이터포털의 해당 활용신청
  상태 또는 서비스 gateway 상태 확인이 필요하다. 이 실패는 다른 provider의 저장을 취소하지 않는다.

## 남은 운영 조건

- `.env`에 `POLICY_BRIEFING_SERVICE_KEY`가 있어야 정책브리핑 API provider가 실행된다.
- 현재 실행 중인 서버는 재시작 후 새 Python 수집·조회 코드를 사용한다.

### 기능 분리 최종 계약 (2026-07-23)

- `오늘 기사 검색`은 기존 언론기사와 국무조정실·기후에너지환경부 직접 수집을 유지한다.
- `정부 보도자료 수집`은 공공데이터포털 정책브리핑 API만 실행한다. 직접 수집이나 GDELT로
  대체하지 않는다.
- 정책브리핑 API가 반환해 저장한 모든 항목은 `governmentPressRelease=true`로 조회되어
  `정부부처 보도자료` 필터에 모두 표시된다.
- 현재 운영 인증키 요청은 HTTP 403을 반환하므로 정책브리핑 API 정상 수집 건수는 0건이다.

### 날짜만 있는 정부자료 경계 회귀 (2026-07-23 09:13 KST)

- 정부 전용 수집은 원본 60건을 받았지만 기존 UTC 시각 비교가 전일 자료를 오전 9시에 일괄
  만료시켜 오늘 자료 1건만 남기는 현상을 재현했다.
- 국무조정실·기후에너지환경부처럼 게시일만 제공되는 provider는 서울 기준 보고일·전일을 허용하고
  2일 전부터 제외하도록 수집·후보 조회 규칙을 통일했다. 시각이 있는 자료의 정확한 24시간 규칙은
  변경하지 않았다.
