# Gemma 브리핑 기사 추천 회귀 기록 — 2026-07-17

## 범위

- 전체 후보 중 규칙 점수 상위 최대 60건을 Gemma 입력으로 구성
- 공사 보도자료 전재·기반 기사를 Gemma 자동 추천 후보에서 제외
- 지역 공공기관의 저심각도 예방 홍보·캠페인을 제외하고 실제 지역 사고는 유지
- 해외 사고는 후보에 유지하되 국내 사고보다 낮은 우선순위로 정렬
- 정부부처·경제·AI 유효 후보를 각각 최소 1건 추천하고 서버에서 충족 여부 검증
- 제목이 재작성된 동일 날짜 공사 보도자료 기반 기사도 자동 추천에서 제외
- 현재 선정 수가 20건에 도달할 때까지 기사 추천
- 추천 생성과 실제 `selected=true` 적용 분리
- 추천 실행·오류·적용 이력 보존
- 기존 수동 선정·메모·중요 표시·Top Issue·분류 override 보존

## 자동 검증

```text
.venv/bin/python -m pytest -q
246 passed, 3 warnings

.venv/bin/ruff check .
All checks passed!

node --check frontend/js/features/auto-selection.js
node --check frontend/js/app.js
성공
```

신규 통합 테스트는 추천만으로 작업본 revision과 선정 상태가 변하지 않는 경로, 명시적
apply, 기존 수동 선정·메모·중요 표시 보존, 존재하지 않거나 중복된 후보 ID의 전체 거부,
migration 생성을 확인한다. 단위 테스트는 최종 출처 판정이 `kesco_republication` 또는
`kesco_based`인 기사가 Gemma 입력 evidence에서 제외되고 독자 취재·미분류 기사는 유지되는지
확인한다. 지역 소방서의 홍보·캠페인은 제외하되 인명피해가 있는 지역 사고와 전국 단위 예방
활동은 유지되는지도 확인한다. 해외 사고는 evidence에서 제거되지 않으면서 동급의 국내 사고보다
뒤로 정렬되는지 확인한다. 상위 60건 밖의 정부부처·경제·AI 후보도 입력에 보존되는지, 기존 수동
선정이 분야 조건을 충족하는지, 필수 분야가 빠진 AI 응답을 거부하는지 확인한다. 제목을 재작성한
실제 사례 형태의 공사 보도자료 기반 기사도 판정한다. 테스트는 외부 Ollama·언론사 네트워크를
호출하지 않는다.

## 수동 회귀

운영 DB와 분리한 임시 SQLite DB와 `127.0.0.1:8790`에서 확인했다.

- [x] `?noauto`로 자동 수집을 막고 기준 화면을 열었다.
- [x] 미선정 후보가 없으면 `Gemma 추천 20건` 버튼이 비활성화된다.
- [x] 미선정 후보가 있으면 버튼이 활성화되고 실행 중 중복 클릭이 차단된다.
- [x] 실제 `gemma4:26b` 실행 뒤 추천 순위·제목·매체·선정 이유·분석 한계가 모달에 표시된다.
- [x] 추천 생성 직후 기사 선정 수는 바뀌지 않는다.
- [x] `추천 적용` 뒤 추천 기사만 선정되고 선정 수·기사 카드·기본 요약이 갱신된다.
- [x] 추천 적용 뒤 기사 메모 저장과 중요 표시가 정상 동작하며 API 재조회에서도 보존된다.
- [x] 인쇄 미리보기가 2페이지로 열리고 추천 적용 기사가 Media Coverage에 포함된다.
- [x] 개발자 콘솔에 기사 추천 모듈의 신규 오류가 없다. 임시 DB에 collection run이 없어
  기존 `GET /api/collections/latest`가 반환한 400 한 건만 표시됐다.

날짜 변경, 요약 직접 수정, JSON·CSV 왕복은 변경 코드와 독립된 기존 경로이며 전체 통합
테스트로 재검증했다. 사용자 Downloads와 운영 DB를 변경하지 않기 위해 임시 브라우저에서
실제 파일 다운로드·재가져오기는 반복하지 않았다.

## 남은 위험

- 후보는 컨텍스트 상한을 지키기 위해 규칙 점수 상위 60건으로 제한한다. 규칙 점수가 낮은
  후보는 Gemma 재평가 대상에서 제외될 수 있다.
- 저장된 전문이 없는 기사는 RSS 요약 또는 제목으로 평가하므로 추천 이유의 정밀도가 낮을
  수 있다. 모달의 분석 한계와 담당자 확인 단계로 이 위험을 노출한다.
- 후보가 20건보다 적거나 기존 수동 선정이 있으면 결과는 정확히 신규 20건이 아니라 전체
  선정 수가 최대 20건이 되도록 부족분만 추가한다.
